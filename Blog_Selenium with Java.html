Selenium with Java: A Complete Tutorial

A step-by-step guide on using Selenium with Java for web automation testing.


Selenium is an open-source tool which is most used in automation testing frameworks for web applications. It supports various programming languages, such as Java, Python, C#, and JavaScript. Java is one of the most popular choices among all, because of its robustness and extensive libraries. This article gives you a step-by-step guide on how to use Selenium with Java, consisting of installation, setup, writing test scripts, and best practices.

Introduction to Selenium

Selenium is a collection of tools and libraries for automating web browsers. It allows testers and developers to write automated test scripts to validate web applications. It can be integrated with other testing frameworks like TestNG, Junit for better organization of test cases. Selenium can run on different OS like Windows, Linux, and macOS. The following are the 4 components of Selenium Suite:

  • Selenium IDE: A browser extension for recording and running scripts.
  • Selenium RC: The main automation tool that directly interacts with web browsers using programming languages.
  • Selenium WebDriver: Upgraded version of Selenium RC, which interacts with web drivers.
  • Selenium Grid: Enables parallel execution of tests across different browsers and machines.

Why Use Java with Selenium?

Java is one of the most preferred languages for Selenium automation due to:

  • Strong community support: Java has extensive documentation and a large developer community.
  • Robust libraries: Java provides powerful libraries like TestNG and JUnit for test execution.
  • Cross-platform compatibility: Java runs on multiple OS without modification.
  • Integration with frameworks: Java easily integrates with frameworks like Maven, Cucumber, and Jenkins.

Prerequisites for using Selenium with Java

Before starting, ensure you have the following installed:

  • Eclipse IDE (Eclipse IDE has a built-in version of JDK, hence no need to install Java separately).
  • Maven
  • Browser drivers (e.g., ChromeDriver for Google Chrome)

Setting Up Selenium with Java (Using Maven)

After installing Eclipse,

  1. Open the Eclipse IDE in your system.
  2. Go to File > New > Maven Project.
  3. Click on Finish to create your project after entering required details.
  4. Add the following dependency to your pom.xml file:
<dependencies>
  <dependency>
     <groupId>org.seleniumhq.selenium</groupId>
      <artifactId>selenium-java</artifactId>
     <version>4.8.0</version>
  </dependency>
</dependencies>

After adding the dependency, update your Maven project to download the required libraries.

Let’s begin with writing a Basic Selenium Test Script in Java

Create a new package in the folder src/main/java in the newly created Maven Project and then create a new class inside that package.

1. Import Required Packages in that class

import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;

2.Initialize WebDriver and Launch Browser

public class SeleniumTest {
  public static void main(String[] args) {
    System.setProperty("webdriver.chrome.driver","**path in which chromedriver is placed**");
    WebDriver driver = new ChromeDriver();

    driver.get("https://www.google.com");
    System.out.println("Page Title: " + driver.getTitle());

    driver.quit();
  }
}

How does it work??

  • System.setProperty() sets the location of the WebDriver.
  • WebDriver driver = new ChromeDriver(); initializes the Chrome browser.
  • driver.get() opens the given URL.
  • driver.getTitle() fetches the webpage title.
  • driver.quit() closes the browser.

Locating Web Elements in Selenium

Selenium provides various locators to identify web elements. They are:

  • ID: driver.findElement(By.id("elementID"));
  • Name: driver.findElement(By.name("elementName"));
  • Class Name: driver.findElement(By.className("className"));
  • CSS Selector: driver.findElement(By.cssSelector("cssSelector"));
  • XPath: driver.findElement(By.xpath("XPathExpression"));
  • Link Text: driver.findElement(By.linkText("linkText"));
  • Partial Link Text: driver.findElement(By.partialLinkText("partialLinkText"));
  • Tag Name: driver.findElement(By.tagName("tagName"));

Code Example: Locating Elements

import org.openqa.selenium.By;
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.WebElement;
import org.openqa.selenium.chrome.ChromeDriver;

public class LocateElement {
  public static void main(String[] args) {
    System.setProperty("webdriver.chrome.driver", "path/to/chromedriver");
    WebDriver driver = new ChromeDriver();
    driver.get("https://www.google.com");

    WebElement searchBox = driver.findElement(By.id("input"));
    searchBox.sendKeys("Search box Located");

    driver.quit();
  }
}

Handling Alerts and Pop-ups in Selenium

Selenium has an in-built Alert interface to handle JavaScript pop-ups or alerts. Given below is a code example on how to handle alerts in Selenium with Java.

import org.openqa.selenium.Alert;
import org.openqa.selenium.By;
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;

public class AlertHandling {
  public static void main(String[] args) {
    System.setProperty("webdriver.chrome.driver", "path/to/chromedriver");
    WebDriver driver = new ChromeDriver();

    driver.get("https://www.selenium.dev/documentation/webdriver/interactions/alerts/");
    driver.findElement(By.xpath("//a[contains(text(),'See an example')]")).click();

    Alert alert = driver.switchTo().alert();
    System.out.println(alert.getText());
    alert.accept();

    driver.quit();
  }
}

Working with Dropdowns in Selenium

To handle dropdown menus, the Select class can be used. Check out the below example for a better understanding of handling dropdowns.

import org.openqa.selenium.By;
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.WebElement;
import org.openqa.selenium.chrome.ChromeDriver;
import org.openqa.selenium.support.ui.Select;

public class DropdownHandling {
  public static void main(String[] args) {
    System.setProperty("webdriver.chrome.driver", "path/to/chromedriver");
    WebDriver driver = new ChromeDriver();
    driver.get("https://www.bstackdemo.com/");

    WebElement dropdownElement = driver.findElement(By.xpath("//select"));
    Select dropdown = new Select(dropdownElement);
    dropdown.selectByVisibleText("Lowest to highest");

    driver.quit();
  }
}

Running Tests with TestNG

TestNG is a testing framework that supports parallel execution and with the help of annotations we can indicate the type of test, pass parameters to test methods, and more.

How to Install TestNG (For Maven Users)

Add the following dependency in pom.xml:

<dependency>
<groupId>org.testng</groupId>
  <artifactId>testng</artifactId>
     <version>7.4.0</version>
   <scope>test</scope>
</dependency>

Writing a TestNG Test Case

import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;
import org.testng.annotations.AfterClass;
import org.testng.annotations.BeforeClass;
import org.testng.annotations.Test;

public class TestNGExample {
  WebDriver driver;

  @BeforeClass
  public void setup() {
    System.setProperty("webdriver.chrome.driver", "path/to/chromedriver");
    driver = new ChromeDriver();
  }

  @Test
  public void openGoogle() {
    driver.get("https://www.google.com");
    System.out.println("Page Title: " + driver.getTitle());
  }

  @AfterClass
  public void quitBrowser() {
    driver.quit();
  }
}
